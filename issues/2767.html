2767. Non-defining declarations of anonymous unions

Section: 11.5.2  [class.union.anon]     Status: open     Submitter: Hubert Tong     Date: 2023-07-06

Currently, the specification permits this example:

  namespace {
    extern union { int x, y; };
    union { int x, y; };
  };

However, those non-defining declarations seem to be mostly useless, because any definition attempt would violate the one-definition rule (6.3 [basic.def.odr]).

Possible resolution:

Change in 11.5.2 [class.union.anon] as follows:

Anonymous unions declared in the scope of a namespace with external linkage shall be declared static. Anonymous unions declared at block scope shall be declared with any storage class allowed for a block variable, or with no storage class. An anonymous union whose declaration inhabits a namespace scope shall have internal linkage (6.6 [basic.link]). An anonymous union whose declaration inhabits a block scope shall be a block variable (6.4.3 [basic.scope.block]). A storage class is not allowed in a declaration of an anonymous union in a class scope.
