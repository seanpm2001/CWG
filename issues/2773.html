2773. Naming anonymous union members as class members

Section: 11.5.2  [class.union.anon]     Status: open     Submitter: Hubert Tong     Date: 2023-07-16

Consider:

  template <typename T>
  int f(T *tp) { return tp->x; }

  static union { int x = f(this); };

According to 11.5.2 [class.union.anon] paragraph 1:

... The names of the members of an anonymous union are bound in the scope inhabited by the union declaration.

Thus, the example above is ill-formed, because the member names are not bound in the scope of the class of the anonymous union. However, there is implementation divergence: clang and gcc accept the example (contrary to the wording), icc rejects. Notwithstanding, this refers to the anonymous union itself, not to an enclosing class.
