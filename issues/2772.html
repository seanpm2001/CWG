2772. Missing Annex C entry for linkage effects of linkage-specification

Section: C.5.4  [diff.cpp03.dcl.dcl]     Status: open     Submitter: Hubert Tong     Date: 2023-07-15

With C++11, extern "C" obtained the effects of extern on linkage of the names declared in the name-declaration of a linkage-specification, but a corresponding Annex C entry is missing.

Possible resolution:

Add a new paragraph in C.5.4 [diff.cpp03.dcl.dcl] as follows:

Affected subclause: 9.11 [dcl.link]
Change: Language linkage applies to names with external linkage only, and names directly declared in a linkage-specification are considered to be declared extern.
Rationale: Alignment with user expectations.
Effect on original feature: Valid C++ 2003 code may violate the one-definition rule (6.3 [basic.def.odr]) in this revision of C++. For example: 
  namespace { extern "C" int x; }           // #1, previously external linkage and C language linkage, now internal linkage and C++ language linkage
  namespace A { extern "C" int x = 42; }    // #2, external linkage and C language linkage
  int main(void) { return x; }
This code is valid in C++ 2003, but #2 is not a definition for #1 in this revision of C++, violating the one-definition rule.
